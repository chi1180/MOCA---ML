# プロジェクト概要

## MOCA---ML

### プロジェクトの目的

本プロジェクトは、過疎地域におけるバス運行の需要予測機械学習システムです。広島県内の過疎地域を対象に、バス停留所ごとの乗客数を予測することで、効率的なバス運行計画の立案を支援します。

### 主な特徴

1. **リアルな過疎地データの模擬生成**
   - 停留所タイプ（乗車地、降車地、ターミナル）に応じた需要パターン
   - 時間帯・曜日・季節・天候による需要変動の再現
   - 過疎地特有の「ゼロ膨張」（乗客がいない時間帯が多い）パターンの実装

2. **包括的な特徴量エンジニアリング**
   - 周期的特徴量（時刻・曜日の循環表現）
   - 時間特徴量（時間帯区分、季節など）
   - 天候特徴量（気温・降水量のカテゴリ化）
   - 相互作用特徴量（複数特徴の組み合わせ）
   - ラグ特徴量（過去の需要データ）

3. **機械学習パイプライン**
   - PyCaret を使用した自動モデル選択
   - 時系列分割による適切な学習・検証
   - モデル評価・可視化ツール

### ユースケース

- **バス運行事業者**: 効率的な運行計画の立案
- **地方自治体**: 公共交通政策の意思決定支援
- **研究者**: 過疎地域の交通需要分析

### 技術スタック

- **言語**: Python 3.10
- **主要ライブラリ**:
  - pandas: データ操作
  - NumPy: 数値計算
  - PyCaret: AutoML フレームワーク
  - TensorFlow: 深層学習（必要に応じて）
  - scikit-learn: 機械学習（PyCaret経由）

### プロジェクト構成

```
space_01/
├── data_generator.py    # ダミーデータ生成ロジック
├── gtr.py              # データ生成実行スクリプト
├── mlm.py              # 機械学習モデル（特徴量生成・学習・評価）
├── Learning.ipynb      # 探索的データ分析・モデル実験ノートブック
├── bus_operation_data.csv  # 生成されたデータセット
├── pixi.toml           # 依存関係管理
└── docs/               # ドキュメント
```

### データフロー

1. **停留所データ取得** (`gtr.py`)
   - 外部APIまたはキャッシュから停留所情報を取得

2. **ダミーデータ生成** (`data_generator.py`)
   - 停留所情報と期間を指定して運行データを生成

3. **特徴量エンジニアリング** (`mlm.py`)
   - 生データから機械学習用特徴量を生成

4. **モデル学習・評価** (`mlm.py`)
   - 複数のモデルを比較し、最適なモデルを選択

5. **予測・可視化**
   - 予測結果の分析と可視化
